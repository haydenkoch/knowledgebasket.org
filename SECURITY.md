# Security Policy

## Reporting a vulnerability

Please do not open public issues for suspected security vulnerabilities.

Use one of these paths instead:

- Open a private GitHub security advisory if you have repository access.
- Contact the maintainer directly through GitHub if you do not.

Include:

- a short description of the issue
- affected paths or features
- reproduction steps or a proof of concept
- impact assessment if known

We will triage reports privately and coordinate a fix before public disclosure when possible.
