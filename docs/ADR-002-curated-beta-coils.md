# ADR-002: Non-Event Coils Operate As Curated Beta

## Status

Accepted

## Context

Funding, Red Pages, Jobs, and Toolbox already exist as public product surfaces, but they do not yet have the same operational depth as Events. Removing them would discard meaningful implementation and product learning. Treating them as fully complete would overstate the product.

## Decision

Keep Funding, Red Pages, Jobs, and Toolbox public, but treat them as curated beta surfaces until shared submission, moderation, editing, and search-indexing tooling reaches parity.

Near-term execution priority:

1. Funding
2. Jobs
3. Red Pages
4. Toolbox

## Consequences

- We preserve visible product progress without pretending every coil is equally mature.
- UX, copy, and documentation should set the right expectations.
- Shared tooling should be built once and reused across non-event coils instead of creating four separate systems.
